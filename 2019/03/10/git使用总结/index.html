git使用总结

2019-03-10 git git 0 评论 字数统计: 1.8k(字) 阅读时长: 6(分)

下面是我整理的常用 Git 命令清单。几个专用名词的译名如下。

  • Workspace：工作区
  • Index / Stage：暂存区
  • Repository：仓库区（或本地仓库）
  • Remote：远程仓库

更多参考地址：https://www.cnblogs.com/chenwolong/p/GIT.html

1. 把本地电脑和github服务器账号相关联
  1. 使用 HTTPS 是需要输入用户名和密码的方式 每次都要输入 输错就把提交命令再执行一次 会重新让你输入

  2. 使用 SSH 可以让本地电脑和github账号相关联 关联了之后就是永久

    1. 理解原理： 使用一个本地秘钥(电脑的身份证) 把你的本地秘钥和服务器github账号相互绑定 把身份证绑定到服务器账号上，你每次提交都会读取你当前的本地的秘钥 再和你本地仓库管理的github服务器仓库的账号里面的秘钥对应，如果是一样的就认为github是认可这台电脑的就允许提交

    2. 在本地电脑生成一个秘钥：在gitbash黑窗里面执行 ssh-keygen -t rsa -C "70255403@qq.com"
      输入后一路回车 出现2048就表示成功

    3. 秘钥生成后会在C盘 -> 用户 -> 你的用户(英文或者administor) -> .ssh -> id_rsa.pub
      id_rsa.pub 本地公钥(公开的钥匙) ,把这个id_rsa.pub 使用记事本打开 把乱码复制一下

    4. 打开github账号

      1. 点击设置
      2. 点击SSH设置
      3. 点击右上角new SSHKey
      4. 输入title(你的名字) key (复制公钥里面的乱码)
      5. 点击Add即可
      6. 出现一个黑色的钥匙就表示成功
2: 常用命令
2.1 设置 git 当前使用者信息

  1. git config --list 显示当前的 git 配置

  2. git config -e [--global] 编辑git 配置文件

  3. git config --global user.email "邮箱"

  4. git config --global user.name "用户名"

  5. C:\Users\Administrator -> .gitconfig 查看当前git的用户名和邮箱是谁 -> 可以直接修改

    ​

2.2 基础命令

  1. git clone 仓库地址 克隆文件到当前文件夹中

  2. git init 初始化仓库，在你的目录会生成 .git 文件夹

  3. git add . / 文件名/ 文件夹 （或者指定文件名或者文件名）

  4. git status 查看暂存区里面有哪些文件，可以查看哪些提交了，哪些没有提交

  5. git commit -m '描述' ‘提交日志,(写上做了些什么,必填)’ 提交日志,把暂存区的文件提交到本地仓库

  6. git log 查看本地仓库所有提交的详细日志

  7. git log --oneline 查看简短的日志

  8. git reflog 查看本地仓库所有版本号

  9. git pull 把远程代码更新

    ​

3: 远程仓库和本地仓库进行关联的命令
  1. git remote add origin 远程仓库地址 指定远程仓库地址,建立连接
  2. git remote rm origin删除远程仓库地址
  3. git push -u orgin master 把本地代码提交到远程仓库主分支中 master主分支 -u指定提交到master分支
  4. git push -u origin 分支名 提交到指定的分支
  5. git push --set-upstream origin master 提交文件到远程仓库
4: 远程关联错误，重新建立关联
  1. git remote 改变远程地址
  2. 如果设置了一个远程仓库地址 但是不是你想要的
  3. 先删除之前的远程仓库地址 git remote rm origin
  4. 再添加一个新的远程仓库地址 git remote add origin 地址
5: 分支
  1. git branch查看分支
  2. git branch 分支名创建分支 -> 必须先提交一次文件(主支)到本地仓库,才能创建分支
  3. git checkout 分支名切换分支
  4. git merge 分支名合并分支 -> 如果当前在分支zhangsan中合并主支-> 把主支文件拷贝到当前分支中->重复的文件名会报错
  5. git branch -d 分支名 删除本地分支
  6. git push -u origin 分支名 提交到指定的分支
6: 回滚撤销
  1. git reset --hard HEAD^ 回到上一个版本
  2. git reset --hard 版本号 回到指定的版本
  3. git diff 查看本地代码和之前仓库代码的区别 （新版本和旧版本代码的区别）
  4. git checkout -- 文件 撤销更改
  5. git reset HEAD 文件 撤销add
  6. git checkout 撤退撤销当前代码的更改
  7. git checkout -- 指定文件.扩展名撤销单个文件修改
  8. git reset HEAD .撤退add了之后才可以撤销某个文件
  9. git checkout -- . 撤销所有修改
  10. git reset HEAD 文件名.拓展名 单独把这个文件撤销
7: 一个项目从头到尾使用git步骤
  1. 创建一个项目
  2. 进入项目打开git bash 黑窗
  3. 输入git ini 初始化一个本地仓库
  4. git add . 把本地代码添加到暂存区
  5. git commit -m '提交日志
  6. 第一次commit可能会报错
    配置用户名 git config --global user.name "xiehao"
    配置邮箱 git config --global user.email "70255403@qq.com"
  7. 配置完成后重新执行 commit 命令 知道出现 create mode表示成功
  8. 如果代码有更新 再次 git add . 再次 git commit -m ‘提交日志’
  9. 创建一个远程仓库
  10. 关联本地和远程仓库
    git remote add origin 远程仓库的地址
    如果关联错了 先删除后关联 git remote rm origin
    再 添加git remote add origin 远程仓库的地址
  11. 把代码提交到远程 git push -u origin master -u表示默认值 默认提交到master分支
    把本地仓库代码提交到服务器里面 git push -u orgin master
  12. 如果是第一次提交没有配置SSH秘钥是无法提交
    1. 黑窗生成一公钥 ssh-keygen -t rsa -C “邮箱” 把邮箱改成自己的
    2. 复制.ssh 里面的.pub公钥里的乱码去github绑定 -> 右上角投降设置->ssh里面绑定秘钥
    3. 使用push 把本地仓库代码提交到远程仓库
    4. 如果后续本地仓库有更新,就必须从新执行下面代码
      1. git add .
      2. git commit -m ‘提交日志’
      3. git push
  1. 一般大项目就会有分支 作用是项目统一 把项目同时的多个版本统一到一起 减少仓库
8: 工作中开发分支的使用
8.1 在侧分支上开发
  1. git init
  2. git add .
  3. git commit -m '描述'
  4. git branch dev创建一个名字为 dev 的分支
  5. 去 github 创建仓库
  6. git remote add origin git@github.com:xiechunhao/hm_manager.git 仓库地址（ssh）本地仓库和远程仓库创建连接
  7. git push -u origin master先提交到主分支
  8. 在 github 上，创建一个分支，名字为 dev
  9. git checkout dev 切换到 dev 分支
  10. 在本地添加一个文件，然后 提交 git add . git commit -m '描述'
  11. git push origin dev就可以提交到 dev 分支上，而不是主分支 master
  12. 测试.txt 文件 只有侧分支有，主分支没有

8.2 侧分支合并到主分支：
  1. git checkout master （切换到主分支）
  2. git merge dev（把某个分支合并到当前所在的分支，这里就是把 dev 分支合并到 master 分支）
